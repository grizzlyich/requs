# Краткая архитектура системы

## Основные сущности
- **Department** — подразделение предприятия.
- **Ticket** — заявка на выполнение работ.
- **Comment** — комментарий к заявке.
- **StatusHistory** — история изменения статусов.

## Пользовательские роли
- **Администратор** — полный доступ.
- **Руководитель** — управление всеми заявками и контроль сроков.
- **Сотрудник** — создание заявок и работа с назначенными задачами.
- **Контролёр** — проверка результатов и участие в контроле качества.

## Основные бизнес-процессы
1. Инициатор создаёт заявку.
2. Руководитель назначает исполнителя и срок.
3. Исполнитель переводит заявку в работу.
4. После выполнения задача переводится в статус «На проверке».
5. Проверяющий завершает заявку или возвращает её на доработку.
